# Prozorro Support Assistant

Короткий огляд курсового проєкту для презентації.

## 1. Що це за проєкт

**Prozorro Support Assistant** — це мультиагентний AI-асистент для підтримки користувачів електронної системи публічних закупівель України.  
Проєкт орієнтований на три типи запитів:

- **юридичні**: норми законодавства, постанови, статті, вимоги;
- **процедурні**: як працюють закупівлі, які є правила та кроки;
- **технічні**: робота з ЕСЗ / майданчиками, типові збої, інструкції, актуальні матеріали.

## 2. Яку проблему вирішує

У користувачів Prozorro питання зазвичай змішані: частина стосується закону, частина процедури, частина технічної поведінки системи.  
Один універсальний бот часто дає поверхневу відповідь, тому тут використано **розподіл ролей між кількома агентами** з окремою перевіркою якості відповіді.

## 3. Як працює система

Система побудована за патернами **Orchestrator-Workers + Evaluator-Optimizer**:

1. **Planner** аналізує запит і визначає, чи він у домені проєкту.
2. Якщо запит коректний, Planner розбиває його на підзадачі.
3. Підзадачі паралельно обробляють профільні агенти:
   - **Lawyer** — працює з нормативною базою;
   - **Common Support** — відповідає на процедурні питання;
   - **Technical Support** — працює з технічними кейсами.
4. Відповіді об’єднуються в одну.
5. **Critic** перевіряє повноту, структуру і актуальність.
6. Якщо відповідь слабка, система запускає **цільове доопрацювання** лише для потрібних агентів.
7. Якщо кейс складний або схожий на баг/інцидент, запит іде на **ескалацію в Slack**.

## 4. Чим проєкт технічно цікавий

- **Мультиагентна архітектура** замість одного загального чат-бота.
- **LangGraph** як керуючий граф із fan-out/fan-in логікою.
- **RAG із двома колекціями знань**:
  - `laws` — для законодавства;
  - `articles` — для інструкцій, FAQ, статей і матеріалів підтримки.
- **Hybrid retrieval**: semantic search + BM25 + reranking.
- **Web search через Tavily** для актуальної зовнішньої інформації.
- **PostgreSQL checkpointer** для сесій і контексту діалогу.
- **Langfuse** для трасування, промптів і спостережуваності.
- **Slack integration** для реального каналу взаємодії з користувачами та ескалацій.

## 5. Поточний стан реалізації

На поточний момент у репозиторії вже реалізовано основний end-to-end сценарій:

- Planner + 3 профільні агенти;
- fan-out обробка multi-topic запитів;
- Critic loop для повторної перевірки й доопрацювання;
- Slack entrypoint;
- Qdrant + Postgres інфраструктура;
- ingestion пайплайн для наповнення бази знань;
- набір unit та e2e тестів;
- evaluation артефакти та скріншоти демо-сценаріїв у `docs/screenshots/`.

Тобто це вже не просто концепт, а **робочий прототип мультиагентної системи підтримки**.

## 6. Що можна показати на демо

Для короткої розповіді колегам найкраще працюють три сценарії:

1. **Юридичний запит**  
   Наприклад: питання про норму закону або постанову.

2. **Змішаний запит**  
   Наприклад: одночасно юридичне пояснення + технічний крок у системі.

3. **Ескалація**  
   Наприклад: користувач описує технічний збій, і система передає кейс експерту.

Окремо варто показати, що система також вміє **відхиляти off-topic запити**, тобто не відповідає на питання поза тематикою закупівель.

## 7. Ключова цінність проєкту

Цей проєкт демонструє, як можна побудувати **керовану, розширювану та більш надійну AI-підтримку** для складної предметної області, де важливі:

- точність;
- пояснюваність;
- розподіл відповідальності між агентами;
- контроль якості перед фінальною відповіддю;
- можливість безпечної ескалації до людини.

## 8. Короткий підсумок

**Prozorro Support Assistant** — це мультиагентний AI-помічник для домену публічних закупівель, який поєднує RAG, web search, критику відповіді та людську ескалацію в одному пайплайні.  
Для курсового проєкту це хороший приклад того, як сучасні LLM-фреймворки можна використати не лише для чат-бота, а для **повноцінної керованої системи підтримки**.
